Online Enrollment System

Assumption Socio-Educational Center

Rationale

The manual, paper-based enrollment system at the Assumption Socio-Educational Center (ASEC) has increasingly shown inefficiencies such as lost documents, delayed processing times, and data management issues. Despite partial modernization efforts through the use of platforms like Jotform, the lack of full system integration continues to burden administrative processes. Recognizing these challenges, the researchers propose the development of an Online Enrollment System. This system aims to provide a centralized, streamlined solution that will significantly reduce manual work, improve data accuracy, and enhance overall operational efficiency. This system will feature online registration, automated record updating, grade and attendance management and notification systems. By transitioning to this digital platform, ASEC can modernize its enrollment processes, ensure smoother transactions, improve accessibility for students and parents, and support school staff with efficient record-keeping and reporting tools.


Ultimately, this system is not just an upgrade in technology; it is a vital step toward providing a faster, more reliable, and more user-friendly enrollment experience that aligns with the school's commitment to delivering quality education and service.

Objectives

This study aims to develop an online enrollment system designed to enhance and streamline the student registration process at Assumption Socio-Educational Center (ASEC). Specifically, this study seeks to:

  1. Manage student enrollment and registration;
  2. Manage student profile and information;
  3. Input, update, and view student grades; and
  4. Generate reports;

Database Architecture

The database architecture that we will utilize is a three tier architecture. Three-tier architecture is a widely used software design model that divides applications into three separate layers: the presentation layer, which handles the user interface; the application layer, where data processing takes place; and the data layer, which manages and stores the application's data.


The three-tier architecture divides an application into the presentation, application, and data tiers. The presentation tier acts as the user interface where users interact with the system, we will use Dart for desktop applications. The application tier, also known as the logic or middle tier, processes data collected from users by applying business rules and may modify information stored in the data tier. Meanwhile, the data tier, sometimes called the database or back-end tier, is responsible for storing and managing information using relational databases. We will utilize Firebase SQL.

Queries

1. Manage Student Enrollment and Registration

Enroll a student in a subject for a specific term and academic year
INSERT INTO enroll (student_id, subj_id, grade, term, academic_year)
VALUES (1, 6, NULL, 'Quarter 1', '2024-2025');

SELECT * FROM enroll;
View all enrolled subjects for a student
SELECT s.subj_name, e.term, e.academic_year
FROM enroll e
JOIN subject s ON e.subj_id = s.subj_id
WHERE e.student_id = 1;

2. Manage Student Profile and Information

View student profile by student ID
SELECT * FROM student WHERE student_id = 1;
Update a student's info (contact number or section)
UPDATE student
SET contact_num = '09179999999', section = 'St. Peter'
WHERE student_id = 1;

SELECT * FROM student WHERE student_id = 1;
List all students in a specific grade and section
SELECT student_id, fname, lname, section
FROM student
WHERE grade_level = 'Grade 4' AND section = 'St. Peter';

3. Input, Update, and View Student Grades

Update a student's grade for a subject and term
UPDATE enroll
SET grade = 91.5
WHERE student_id = 1 AND subj_id = 5 AND term = 'Quarter 1';
View all grades of a student
SELECT s.subj_name, e.grade, e.term, e.academic_year
FROM enroll e
JOIN subject s ON e.subj_id = s.subj_id
WHERE e.student_id = 1;

4. Generate Reports

Generate list of students enrolled in a given academic year and term
SELECT DISTINCT st.student_id, st.fname, st.lname, st.grade_level, st.section
FROM enroll e
JOIN student st ON e.student_id = st.student_id
WHERE e.academic_year = '2024-2025' AND e.term = 'Quarter 1';
Generate subject enrollment count per grade level
SELECT gs.grade_level, s.subj_name, COUNT(e.student_id) AS enrolled_students
FROM grade_subject gs
JOIN subject s ON gs.subj_id = s.subj_id
LEFT JOIN enroll e ON gs.subj_id = e.subj_id AND e.academic_year = '2024-2025'
GROUP BY gs.grade_level, s.subj_name;
